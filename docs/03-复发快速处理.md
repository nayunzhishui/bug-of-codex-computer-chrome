# 03-复发快速处理

## 一句话判断

如果 Chrome 扩展已经 Connected，但 Codex 仍不能打开网页，优先查运行时，不要重装 Chrome 扩展。

---

## 先查日志

运行：

```powershell
.\scripts\check-codex-runtime.ps1
```

重点看：


## 快速决策表

| 现象 | 判断 | 处理 |
|---|---|---|
| 插件市场没有 Browser / Chrome / Computer Use | openai-bundled 缺失 | 恢复 bundled marketplace |
| Chrome 扩展 Disconnected | Native Host 未注册 | 修 manifest 和注册表 |
| Chrome Connected 但网页打不开 | 运行时缺失 | 查 node_repl / cua_node / codex.exe / rg.exe |
| 所有网站被策略阻止 | nodeRepl.config / CODEX_CLI_PATH 问题 | 修 CLI 路径 |
| 提示 sandbox setup 缺失 | 沙箱 helper 缺失 | 补 codex-windows-sandbox-setup.exe |
| 提示 command runner 缺失 | 命令执行 helper 缺失 | 补 codex-command-runner.exe |
| 运行时文件都存在但旧线程仍没有 `mcp__node_repl__js` | 旧线程工具表未注入 / 旧 CLI 线程 | 看 `docs/05-旧线程不注入-node-repl.md` |
| 新建线程仍提示无法连接 Chrome 控制组件 | Desktop 仍由旧本地 `codex.exe` 启动，或 `node_repl` MCP 配置丢失 | 看 `docs/06-新线程仍不注入-node-repl.md` |
| 新任务是新 CLI，但 `mcp__node_repl__js` 仍不存在 | bundled marketplace/cache 仍旧，或旧 Native Host 回写配置 | 看 `docs/08-Codex更新后插件缓存仍是旧版.md` |
| 工具先可用，导航后变成 `not a function` | Desktop 使用了与当前 MSIX 不同的 npm 核心二进制 | 检查 `Desktop core alignment`；从当前 MSIX 重同步 |
| 工具存在但报 `Browser security unavailable outside node repl` | 普通外部 stdio MCP 缺少官方可信上下文 | 删除旧 MCP workaround，刷新并重装当前 bundled 插件 |
| `node_repl/js` 已启动，但提示 `Browser is not available: extension` | Google Chrome 未运行，或扩展后端未注册 | 用 `@chrome` 时启动 Chrome；若接受 Edge，改走 Computer Use / 内置 Browser |
| 设置已允许，但提示 `elicitations are unavailable` | CLI / node_repl / node_modules 版本错配，受信任授权桥缺失 | 看 `docs/07-授权已开启但-Computer-Use-仍失败.md` |
| Chrome Connected、能读标签页，但导航被安全层拦截 | 连接正常，属于浏览器安全导航 | 用用户明确给出的完整 URL 复测，不绕过安全层 |

## 不要做

```text
不要先重装 Chrome 扩展
不要先清空 .codex
不要先删除 Chrome 用户数据
不要先反复点“重新安装工作空间依赖项”
不要复用旧 fingerprint
不要在运行时已存在时继续重装 Chrome 扩展
```

## 线程不注入时的最短处理

如果插件和运行时都正常，但当前线程仍不暴露 `mcp__node_repl__js`，先比较版本：

```powershell
codex plugin list
Get-AppxPackage OpenAI.Codex | Select-Object Version, InstallLocation
```

如果活动 marketplace/cache 低于当前 MSIX，运行 dry-run：

```powershell
.\scripts\repair-codex-runtime-skew.ps1
```

确认后完全退出 Codex并关闭 Chrome / Edge 窗口，再执行（脚本会停止残留后台进程）：

```powershell
.\scripts\repair-codex-runtime-skew.ps1 -Apply
```

如果已经新建线程仍失败，检查最新 session 日志：

```powershell
rg -n "无法连接 Chrome 控制组件|cli_version|mcp: node_repl/js started" `
  "$env:USERPROFILE\.codex\sessions"
```

若失败线程显示 `cli_version` 仍是旧版，或 `tools = null`，不要重装 Chrome 扩展；优先修 Desktop 本地运行时和 bundled 插件版本链。

若工具已经调用，但出现 `outside node repl` 或 `elicitations are unavailable`，不要继续围绕“是否点了允许”排查，也不要添加普通外部 `node_repl` MCP。应比较当前 MSIX、活动 marketplace、插件缓存、顶层 `node_repl.exe` 和 Desktop CLI 是否属于同一版本链。
