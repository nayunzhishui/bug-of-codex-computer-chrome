# 03-复发快速处理

## 一句话判断

如果 Chrome 扩展已经 Connected，但 Codex 仍不能打开网页，优先查运行时，不要重装 Chrome 扩展。

---

## 先查日志

运行：

```powershell
.\scripts\check-codex-runtime.ps1
```

重点看：

```text
bundled_executable_relocation_failed
node-repl-missing
missingHelperPath
browser_use_setup_failed
mcp__node_repl__js
codex-windows-sandbox-setup.exe
codex-command-runner.exe
CODEX_CLI_PATH
nodeRepl.config
```

---

## 快速决策表

| 现象 | 判断 | 处理 |
|---|---|---|
| 插件市场没有 Browser / Chrome / Computer Use | openai-bundled 缺失 | 恢复 bundled marketplace |
| Chrome 扩展 Disconnected | Native Host 未注册 | 修 manifest 和注册表 |
| Chrome Connected 但网页打不开 | 运行时缺失 | 查 node_repl / cua_node / codex.exe / rg.exe |
| 所有网站被策略阻止 | nodeRepl.config / CODEX_CLI_PATH 问题 | 修 CLI 路径 |
| 提示 sandbox setup 缺失 | 沙箱 helper 缺失 | 补 codex-windows-sandbox-setup.exe |
| 提示 command runner 缺失 | 命令执行 helper 缺失 | 补 codex-command-runner.exe |
| 运行时文件都存在但旧线程仍没有 `mcp__node_repl__js` | 旧线程工具表未注入 / 旧 CLI 线程 | 看 `docs/05-旧线程不注入-node-repl.md` |
| 新建线程仍提示无法连接 Chrome 控制组件 | Desktop 仍由旧本地 `codex.exe` 启动，或 `node_repl` MCP 配置丢失 | 看 `docs/06-新线程仍不注入-node-repl.md` |
| 设置已允许，但提示 `elicitations are unavailable` | CLI / node_repl / node_modules 版本错配，受信任授权桥缺失 | 看 `docs/07-授权已开启但-Computer-Use-仍失败.md` |
| Chrome Connected、能读标签页，但导航被安全层拦截 | 连接正常，属于浏览器安全导航 | 用用户明确给出的完整 URL 复测，不绕过安全层 |

## 不要做

```text
不要先重装 Chrome 扩展
不要先清空 .codex
不要先删除 Chrome 用户数据
不要先反复点“重新安装工作空间依赖项”
不要复用旧 fingerprint
不要在运行时已存在时继续重装 Chrome 扩展
```

## 旧线程不注入时的最短处理

如果插件和运行时都正常，但当前线程仍不暴露 `mcp__node_repl__js`：

```powershell
$codexCli = "$env:LOCALAPPDATA\OpenAI\Codex\bin\codex.exe"
$nodeRepl = "$env:LOCALAPPDATA\OpenAI\Codex\bin\node_repl.exe"
$mods = "$env:LOCALAPPDATA\OpenAI\Codex\runtimes\cua_node\<当前版本>\bin\node_modules"
codex mcp add node_repl `
  --env CODEX_CLI_PATH=$codexCli `
  --env CODEX_HOME="$env:USERPROFILE\.codex" `
  --env NODE_REPL_NODE_MODULE_DIRS=$mods `
  -- $nodeRepl
codex mcp list
```

然后新 turn / 新线程 / 重启 Codex 后复测：

```text
tool_search("node_repl js")
```

预期应出现：

```text
mcp__node_repl__js
```

如果已经新建线程仍失败，检查最新 session 日志：

```powershell
rg -n "无法连接 Chrome 控制组件|cli_version|mcp: node_repl/js started" `
  "$env:USERPROFILE\.codex\sessions"
```

若失败线程显示 `cli_version` 仍是旧版，或 `tools = null`，不要重装 Chrome 扩展；优先修 Desktop 本地运行时和显式 `node_repl` MCP。

若工具已经调用，但出现 `elicitations are unavailable`，不要继续围绕“是否点了允许”排查。应比较当前 MSIX manifest、顶层 `node_repl.exe`、`NODE_REPL_NODE_MODULE_DIRS` 和 Desktop CLI 是否属于同一版本链。
