# 04-日志关键词表

| 关键词 | 含义 | 下一步 |
|---|---|---|
| `bundled_executable_relocation_failed` | WindowsApps 内置执行文件迁移失败 | 查源路径和目标路径，文件流复制 |
| `node-repl-missing` | `node_repl.exe` 缺失或目标路径错误 | 查 `cua_node` fingerprint |
| `missingHelperPath` | helper 路径缺失 | 查 RuntimePaths |
| `browser_use_setup_failed` | Browser Use 初始化失败 | 查 node_repl / sandbox |
| `mcp__node_repl__js` | JavaScript 执行入口缺失 | 查 node_repl 与 CODEX_CLI_PATH |
| `CODEX_CLI_PATH` | Codex CLI 环境变量 | 应指向版本化 codex.exe |
| `nodeRepl.config` | 权限桥配置 | 若缺失，检查本地 CLI |
| `codex-windows-sandbox-setup.exe` | Windows 沙箱初始化 helper | 从安装包补齐 |
| `codex-command-runner.exe` | 命令执行 helper | 从安装包补齐 |
| `enterprise network policy` | 可能是权限桥失败，不一定是真网络限制 | 先查 nodeRepl.config |
| `Disconnected` | Chrome 扩展未连 Native Host | 查 manifest 和注册表 |
| `Connected` | Chrome 扩展到 Native Host 已通 | 继续查运行时 |

## 判断原则

```text
如果 Chrome 扩展已经 Connected，就不要继续围绕 Chrome 扩展排查。
如果日志继续出现 node_repl、RuntimePaths、sandbox 或 command runner，就优先修运行时。
如果 Codex 更新后复发，先读取新 hash，不要复用旧路径。
```
