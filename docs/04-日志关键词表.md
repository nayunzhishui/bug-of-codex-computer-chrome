# 04-日志关键词表

| 关键词 | 含义 | 下一步 |
|---|---|---|
| `bundled_executable_relocation_failed` | WindowsApps 内置执行文件迁移失败 | 查源路径和目标路径，文件流复制 |
| `node-repl-missing` | `node_repl.exe` 缺失或目标路径错误 | 查 `cua_node` fingerprint |
| `missingHelperPath` | helper 路径缺失 | 查 RuntimePaths |
| `browser_use_setup_failed` | Browser Use 初始化失败 | 查 node_repl / sandbox |
| `mcp__node_repl__js` | JavaScript 执行入口缺失 | 查 node_repl 与 CODEX_CLI_PATH |
| `CODEX_CLI_PATH` | Codex CLI 环境变量 | 应指向 Desktop 实际使用且已验证版本一致的 codex.exe；新旧包路径不同 |
| `nodeRepl.config` | 权限桥配置 | 若缺失，检查本地 CLI |
| `codex-windows-sandbox-setup.exe` | Windows 沙箱初始化 helper | 从安装包补齐 |
| `codex-command-runner.exe` | 命令执行 helper | 从安装包补齐 |
| `enterprise network policy` | 可能是权限桥失败，不一定是真网络限制 | 先查 nodeRepl.config |
| `Disconnected` | Chrome 扩展未连 Native Host | 查 manifest 和注册表 |
| `Connected` | Chrome 扩展到 Native Host 已通 | 继续查运行时 |
| `thread_dynamic_tools` | 线程动态工具表 | 若为空且旧线程无 `mcp__node_repl__js`，查旧线程注入 |
| `cli_version` | 线程创建时 CLI 版本 | 若低于当前运行时，旧线程可能不热加载新工具 |
| `history_mode=legacy` | 旧线程历史模式 | 可能伴随工具表不刷新 |
| `codex mcp list` | 外部 MCP 配置 | 若出现手工注册的 `node_repl`，警惕其遮蔽官方可信工具 |
| `tools = null` | session meta 没有记录工具表 | 新线程也可能未注入 node_repl |
| `mcp: node_repl/js started` | node_repl 实际被调用 | Chrome / Browser 控制链进入真实执行阶段 |
| `无法连接 Chrome 控制组件` | Chrome skill 找不到控制入口后的兜底提示 | 先查 node_repl 注入，不要直接重装扩展 |
| 桌面内置 `codex.exe` 版本低于 `codex --version` | Desktop 仍在使用旧运行时 | 完全退出后替换 `%LOCALAPPDATA%\OpenAI\Codex\bin` 中的旧 exe |
| `Computer Use requires app approval but elicitations are unavailable` | 已到达逐应用审批阶段，但任务缺少 `createElicitation` 授权桥 | 比较 CLI、node_repl、CUA manifest 和 MCP 模块目录 |
| `Computer Use app approval UI is unavailable outside trusted node_repl` | 当前是普通/不完整 node_repl 环境，不是受信任桌面桥 | 恢复同版本 `config`、`nativePipe`、`createElicitation`，不要自动批准 |
| `Browser security unavailable outside node repl` | 普通外部 MCP 没有官方可信浏览器上下文 | 删除 workaround，比较并刷新 MSIX/marketplace/cache 插件版本 |
| `node_repl_archive_path` | 当前 MSIX 声明的 node_repl 版本 | 用 manifest 动态确定目标运行时，禁止照抄旧版本 |
| `runtime_archive_version` | 当前 CUA 整套运行时版本 | node_repl 与 node_modules 应来自这一套运行时 |
| 导航被浏览器安全层拦截 | Chrome 已连接但安全策略拒绝当前导航 | 用用户明确提供的完整 URL 复测，不绕过安全层 |

## 判断原则

```text
如果 Chrome 扩展已经 Connected，就不要继续围绕 Chrome 扩展排查。
如果日志继续出现 node_repl、RuntimePaths、sandbox 或 command runner，就优先修运行时。
如果 Codex 更新后复发，先读取新 hash，不要复用旧路径。
如果运行时已存在但当前线程仍没有 mcp__node_repl__js，不要反复修 Chrome 扩展；检查旧线程工具表和 bundled 插件版本，不要新增普通外部 node_repl MCP。
如果新建线程仍没有 mcp__node_repl__js，检查 session_meta 的 cli_version、tools 字段，以及 Desktop 本地 codex.exe 版本。
如果已经出现 mcp: node_repl/js started，但 Computer Use 报 elicitations unavailable，停止排查工具发现，转查授权桥和运行时一致性。
```
