# 02-关键修复流程

## 1. 先判断是否还需要修 Chrome 扩展

如果 Chrome 扩展仍是 `Disconnected`，检查 Native Host。  
如果已经 `Connected`，不要继续修 Chrome 扩展，直接查运行时。

---

## 2. 检查 openai-bundled

执行：

```powershell
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\.agents\plugins\marketplace.json"
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\browser\.codex-plugin\plugin.json"
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\chrome\.codex-plugin\plugin.json"
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\computer-use\.codex-plugin\plugin.json"
```

如果都为 `True`，不要再修这一层。

---

## 3. 检查 Native Host

执行：

```powershell
$hostName = "com.openai.codexextension"
$manifest = "$env:LOCALAPPDATA\OpenAI\extension\$hostName.json"

Test-Path $manifest
reg query "HKCU\Software\Google\Chrome\NativeMessagingHosts\$hostName" /ve
```

如果 Chrome 扩展已经 Connected，不要再修这一层。

---

## 4. 查运行时日志

运行：

```powershell
.\scripts\check-codex-runtime.ps1
```

重点看：

```text
bundled_executable_relocation_failed
node-repl-missing
missingHelperPath
browser_use_setup_failed
mcp__node_repl__js
codex-windows-sandbox-setup.exe
codex-command-runner.exe
CODEX_CLI_PATH
nodeRepl.config
```

---

## 5. 读取版本化目标路径

必须从日志中提取：

```text
CodexFingerprint
RgFingerprint
CuaFingerprint
RuntimePaths
```

不要手动猜路径。

本次案例：

```text
CodexFingerprint：38dff8711e296435
RgFingerprint：ada252862d154cdd
CuaFingerprint：1b23c930bdf84ed6
```

---

## 6. 使用文件流复制

如果源文件存在，但普通复制失败，不要继续反复复制。  
应改用文件流复制，只复制文件内容，不继承特殊属性。

模板：

```text
scripts/copy-plain-file-template.ps1
```

原则：

```text
只复制内容
不复制特殊属性
校验 SHA-256
目标目录先备份
先写临时目录，再移动
```

---

## 7. 设置 CODEX_CLI_PATH

检查：

```powershell
$cli = [Environment]::GetEnvironmentVariable("CODEX_CLI_PATH", "User")
Write-Host $cli
Test-Path $cli
```

应指向：

```text
%LOCALAPPDATA%\OpenAI\Codex\bin\<CodexFingerprint>\codex.exe
```

本次案例：

```text
%LOCALAPPDATA%\OpenAI\Codex\bin\38dff8711e296435\codex.exe
```

设置后必须完全退出 Codex 并重启电脑。

---

## 8. 处理 sandbox / command-runner

如果日志显示缺少：

```text
codex-windows-sandbox-setup.exe
codex-command-runner.exe
```

只补这两个文件。  
不要重新处理 Chrome 扩展、Native Host、openai-bundled。

---

## 9. 最终验证

重启后只打开官方 Codex，新建线程测试：

```text
@Chrome 打开 https://example.com 并告诉我页面标题
@Browser 打开 https://example.com 并告诉我页面标题
使用电脑操控打开记事本，并告诉我是否成功
```
