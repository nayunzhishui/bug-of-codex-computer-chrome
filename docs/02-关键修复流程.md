# 02-关键修复流程

## 1. 先判断是否还需要修 Chrome 扩展

如果 Chrome 扩展仍是 `Disconnected`，检查 Native Host。  
如果已经 `Connected`，不要继续修 Chrome 扩展，直接查运行时。

---

## 2. 检查 openai-bundled

执行：

```powershell
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\.agents\plugins\marketplace.json"
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\browser\.codex-plugin\plugin.json"
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\chrome\.codex-plugin\plugin.json"
Test-Path "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\computer-use\.codex-plugin\plugin.json"
```

如果都为 `True`，还要比较这些 manifest 的 `version` 与当前 MSIX 内版本。文件存在但版本旧，仍属于故障。

---

## 3. 检查 Native Host

执行：

```powershell
$hostName = "com.openai.codexextension"
$manifest = "$env:LOCALAPPDATA\OpenAI\extension\$hostName.json"

Test-Path $manifest
reg query "HKCU\Software\Google\Chrome\NativeMessagingHosts\$hostName" /ve
```

如果 Chrome 扩展已经 Connected，不要再修这一层。

---

## 4. 查运行时日志

运行：

```powershell
.\scripts\check-codex-runtime.ps1
```

重点看：

```text
bundled_executable_relocation_failed
node-repl-missing
missingHelperPath
browser_use_setup_failed
mcp__node_repl__js
codex-windows-sandbox-setup.exe
codex-command-runner.exe
CODEX_CLI_PATH
nodeRepl.config
```

---

## 5. 读取当前运行时目标路径

旧包先从日志中提取：

```text
CodexFingerprint
RgFingerprint
CuaFingerprint
RuntimePaths
```

新包还必须读取当前 MSIX 的 `app\resources\cua_node\manifest.json`。不要手动猜路径，也不要假设所有版本都继续使用 fingerprint 目录。

本次案例：

```text
CodexFingerprint：38dff8711e296435
RgFingerprint：ada252862d154cdd
CuaFingerprint：1b23c930bdf84ed6
```

---

## 6. 使用文件流复制

如果源文件存在，但普通复制失败，不要继续反复复制。  
应改用文件流复制，只复制文件内容，不继承特殊属性。

模板：

```text
scripts/copy-plain-file-template.ps1
```

原则：

```text
只复制内容
不复制特殊属性
校验 SHA-256
目标目录先备份
先写临时目录，再移动
```

---

## 7. 设置 CODEX_CLI_PATH

检查：

```powershell
$cli = [Environment]::GetEnvironmentVariable("CODEX_CLI_PATH", "User")
Write-Host $cli
Test-Path $cli
```

应指向当前 Desktop 实际使用、且已验证版本一致的 `codex.exe`。常见位置包括：

```text
%LOCALAPPDATA%\OpenAI\Codex\bin\codex.exe
%LOCALAPPDATA%\OpenAI\Codex\bin\<CodexFingerprint>\codex.exe   # 旧包
```

早期案例：

```text
%LOCALAPPDATA%\OpenAI\Codex\bin\38dff8711e296435\codex.exe
```

设置后必须完全退出 Codex/ChatGPT 并重新打开。通常不需要重启整台电脑；只有进程无法释放旧文件时才考虑重启系统。

---

## 8. 处理 sandbox / command-runner

如果日志显示缺少：

```text
codex-windows-sandbox-setup.exe
codex-command-runner.exe
```

只补这两个文件。  
不要重新处理 Chrome 扩展、Native Host、openai-bundled。

---

## 9. 最终验证

重启后只打开官方 Codex，新建线程测试：

```text
@Chrome 打开 https://example.com 并告诉我页面标题
@Browser 打开 https://example.com 并告诉我页面标题
使用电脑操控打开记事本，并告诉我是否成功
```

---

## 10. 运行时存在但线程仍没有 mcp__node_repl__js

如果以下条件同时成立：

```text
browser@openai-bundled installed, enabled
chrome@openai-bundled installed, enabled
computer-use@openai-bundled installed, enabled
node_repl.exe 存在
chrome-native-hosts-v2.json 存在
tool_search("node_repl js") 仍不返回 mcp__node_repl__js
```

先检查当前线程是否是旧线程，并比较 bundled 插件版本：

```powershell
sqlite3 "$env:USERPROFILE\.codex\state_5.sqlite" `
  "select id, cli_version, history_mode, source, model from threads order by updated_at desc limit 10;"

sqlite3 "$env:USERPROFILE\.codex\state_5.sqlite" `
  "select count(*) from thread_dynamic_tools;"
```

```powershell
$package = Get-AppxPackage OpenAI.Codex | Sort-Object Version -Descending | Select-Object -First 1
$msixPlugin = Join-Path $package.InstallLocation 'app\resources\plugins\openai-bundled\plugins\chrome\.codex-plugin\plugin.json'
$activePlugin = "$env:USERPROFILE\.codex\.tmp\bundled-marketplaces\openai-bundled\plugins\chrome\.codex-plugin\plugin.json"
(Get-Content $msixPlugin -Raw | ConvertFrom-Json).version
(Get-Content $activePlugin -Raw | ConvertFrom-Json).version
```

如果版本不同，不能通过添加显式 MCP server 处理。普通 stdio `node_repl` 可能让工具可见，却缺少官方可信浏览器和审批上下文。

先 dry-run：

```powershell
.\scripts\repair-codex-runtime-skew.ps1
```

完全退出 Codex、Chrome 和 Edge 后执行：

```powershell
.\scripts\repair-codex-runtime-skew.ps1 -Apply
```

注意：

```text
修复脚本会移除旧的外部 node_repl workaround，刷新当前 MSIX marketplace，重装同版本 bundled 插件，并更新 Native Host。
应用后必须完全重启 Codex，用新任务复测。
不要因此回头删除 .codex、Chrome 用户数据或重装 Chrome 扩展。
```

若出现 `outside node repl` 或 `elicitations are unavailable`，继续执行下一节并检查 `docs/08-Codex更新后插件缓存仍是旧版.md`。

---

## 11. 授权已开启但 Computer Use 仍失败

精确搜索：

```powershell
rg -n "Computer Use requires app approval|elicitations are unavailable" `
  "$env:USERPROFILE\.codex\sessions"
```

然后比较失败任务、Desktop 本地 CLI 和终端 CLI：

```powershell
& "$env:LOCALAPPDATA\OpenAI\Codex\bin\codex.exe" --version
codex --version
```

再从当前 MSIX manifest 动态读取 `node_repl_archive_path` 与 `runtime_archive_version`。不要继续使用旧 `<CuaFingerprint>`。

先 dry-run：

```powershell
.\scripts\repair-codex-runtime-skew.ps1
```

完全退出 Codex/ChatGPT 后执行：

```powershell
.\scripts\repair-codex-runtime-skew.ps1 -Apply
```

不要修改 helper 自动接受审批。

---

## 12. 区分 Chrome 连接故障和安全导航

如果 Chrome 已 Connected 且能读取标签页，用用户明确提供的完整 URL 复测：

```text
@Chrome 打开 https://example.com
```

站点名称被解析后触发的安全拦截，不应通过修改插件绕过。
